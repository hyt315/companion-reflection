# 贡献指南 / Contributing Guide

感谢你考虑为 Companion Reflection 贡献内容！这个项目旨在为情绪陪伴对话提供高质量的引导框架，你的参与能帮助更多人。

## 贡献方式

- **改进回应方式**：优化现有四种回应模式的措辞、引导逻辑或适用边界
- **新增回应场景**：为尚未覆盖的情绪信号（如愤怒、孤独、失落等）设计新的回应框架
- **补充理论来源**：在 `references/methods-background.md` 中添加新的学术引用或临床依据
- **翻译优化**：改进中英文内容的自然度和准确性
- **报告问题**：发现安全协议漏洞、不当表述或技术错误

## 开发流程

1. **Fork** 本仓库
2. 创建你的特性分支：`git checkout -b feature/your-feature`
3. 做出改动并提交：`git commit -m "feat: add something"`
4. 推送到分支：`git push origin feature/your-feature`
5. 创建 Pull Request

## 提交规范

本项目遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

- `feat:` 新功能或新回应场景
- `fix:` 修复错误表述或安全协议漏洞
- `docs:` 文档改进
- `refactor:` 内容重构（不改变功能）

## 内容准则

- **安全协议是红线**：任何改动不得削弱安全协议的优先级和有效性
- **循证原则**：新增的回应方式应有成熟的心理学或哲学框架支撑
- **坦诚透明**：保持"方法可以不说出来，但不应该被刻意隐藏"的核心原则
- **不做诊断**：所有内容不得包含诊断性或处方性建议

## 审查标准

PR 审查时我们会关注：

- [ ] 内容保持心理学/哲学框架的准确性
- [ ] 不修改安全协议的核心逻辑（除非是增强）
- [ ] 中英文内容结构对应
- [ ] 不修改 LICENSE
- [ ] 回应示例自然、温暖，不说教

## 行为准则

参与本项目即表示你同意遵守 [行为准则](CODE_OF_CONDUCT.md)。
